Pythonのスクリプトの書き方とコメント／docstring

1. はじめに

Pythonプログラミングを学ぶ上で、コードを単に書くだけでなく、誰が読んでも理解しやすく、後から見返したときにも保守しやすい形で残すことは非常に重要です。この記事では、Pythonコードをファイルに保存して実行する「スクリプト」の基本的な書き方と、コードの可読性を高めるための「コメント」および「docstring」の使い方について解説します。

これらのテクニックを身につけることで、あなたのコードはよりプロフェッショナルで、他の開発者と協力しやすくなります。

2. 具体的な使い方や概念の説明

Pythonスクリプトの作成と実行

Pythonスクリプトは、単純なテキストファイルです。拡張子を .py にして保存することで、Pythonのプログラムとして認識されます。

ステップ1: ファイルの作成 まず、hello.py という名前のファイルを作成し、以下のコードを記述します。

# hello.py
print("Hello, World!")

name = "Alice"
print(f"こんにちは、{name}さん！")

ステップ2: スクリプトの実行 作成したファイルは、ターミナル（コマンドプロンプト）から python コマンドを使って実行します。

$ python hello.py

実行結果:

Hello, World!
こんにちは、Aliceさん！

このように、複数の処理をファイルにまとめて記述し、一括で実行するのがスクリプトの基本的な使い方です。

コメントでコードを説明する

コメントは、Pythonのインタプリタに無視されるテキストで、コード内にメモを残すために使用します。コードが何をしているのか、なぜそのような記述をしたのかを説明するのに役立ちます。

コメントは # 記号を使って記述します。

一行コメント # からその行の終わりまでがコメントになります。

# この行はコメントです。プログラムの実行には影響しません。

# ユーザーに挨拶を表示する
print("Hello, Python!")

インラインコメント コードと同じ行に記述することもできます。短い注釈を入れたい場合に便利です。

x = 10  # xはユーザーのスコアを保持する変数

コメントは、複雑なロジックや、一読しただけでは意図が分かりにくいコードに対して「なぜ」こう書いたのかを説明するために使うのが効果的です。

docstringでドキュメントを記述する

docstring（ドキュメンテーション文字列）は、モジュール、関数、クラス、メソッドの定義の直後に置かれる特別なコメントです。"""（トリプルクォート）で囲んで記述します。

docstringの目的は、そのコードブロックが「何をするのか」を説明することです。これは、コードの仕様書のような役割を果たします。

関数のdocstringの例:

def add(a, b):
    """二つの数値を受け取り、その合計を返す関数です。

    Args:
        a (int or float): 一つ目の数値
        b (int or float): 二つ目の数値

    Returns:
        int or float: 二つの数値の合計
    """
    return a + b

# docstringの確認
print(add.__doc__)

docstringの利点: 1. help()関数で参照できる: Pythonの組み込み関数 help() を使うと、その関数のdocstringを整形して表示できます。

```python
help(add)
```

これにより、対話環境で関数の使い方をすぐに確認できます。
  1. ドキュメント自動生成ツールが利用できる: Sphinx のようなツールがdocstringを読み取り、プロジェクトの公式ドキュメントを自動で生成してくれます。

コメントがコードの「実装の詳細」を説明するのに対し、docstringはコードの「使い方や目的」を説明するもの、と覚えておくと良いでしょう。

3. まとめ

今回は、Pythonの基本的なコーディングスタイルについて学びました。

  • スクリプト: .py ファイルにコードを記述し、まとめて実行する方法。
  • コメント (#): コードの意図や複雑な部分を説明するためのメモ。
  • docstring ("""..."""): 関数やクラスが何をするものなのかを説明する公式なドキュメント。
